# KET 单词大冒险

面向 A2 Key（KET）学习者的趣味背单词网页游戏。项目包含 1629 张全量单词卡、每日任务、卡牌收集与分解、三波卡牌对战，以及一次性战斗技能系统。

## 扣子兼容与记忆系统修复（v5）

- 1629 张卡牌改为标准 JSON：`assets/cards-data-v5.json`，启动后先校验数量，再加载游戏逻辑。
- 完全移除 `CARDS_DATA` 和旧 `cards-data.js` 依赖；资源失败时显示可重试的错误卡片，不再只显示蓝色背景。
- 构建标识固定为 `ket-v5-srs-20260721`，便于确认扣子是否真正部署了新版本。
- 部署后网络面板必须看到 `cards-data-v5.json?v=5-srs`，控制台必须出现 `[KET BUILD] ket-v5-srs-20260721 · Loaded 1629 cards`。
- 如果线上仍请求 `cards-data.js` 或出现 `CARDS_DATA is not defined`，说明扣子仍在运行旧构建，需要清理构建缓存并重新部署本压缩包。

## 间隔复习规则

- 每个学习日固定安排 6 个未学新词，同一天多次进入会继续同一份计划，不会重新计数。
- 当天任务完成后，按钮切换为“开始巩固挑战”；巩固答题不会重复推进地图。
- 到期旧词与新词交替出现，复习间隔依次为 1、3、7、14、30、60 天。
- 答错的词会在当前练习的 3–5 题后再次出现，并安排次日复习。
- 地图只按“首次学会的不同单词数”推进，每 6 个不同新词前进 1 关。
- “已学习”和“长期掌握”分开统计；完成 7 天阶段后计入长期掌握。
- 旧存档会自动迁移。旧版被重复累计的掌握量最多按 6 个真实单词迁移，避免地图继续虚高。

## 已实现功能

- 1629 个 KET 词汇及全量卡牌图鉴。
- 图鉴每页 24 张，支持中英文搜索、稀有度筛选、已获得/未获得/重复卡筛选和页码跳转。
- 重复卡牌显示持有数量；分解时至少保留 1 张，按 N/R/SR/SSR 奖励 10/20/35/60 星星。
- 每日任务每日自动重置：首次学会 6 个不同新词、完成到期复习、获得 2 张卡牌、完成 1 场对战，并实时显示进度。
- 每日奖励：20 星星、2 钻石、30 星星；三项全部完成额外获得 3 钻石。
- 冒险地图根据实际首次学会的不同词数推进，新用户从 1-1 开始，重复练习不推进地图。
- 三波卡牌对战、5 张出战卡组、4 个去重释义选项、能量/护盾/连击/敌人意图与战斗特效。
- 胜利按剩余生命、最高连击和零失误表现结算 3–7 钻石。
- 5 钻石随机抽取一次性技能：回血、复活、免伤、开局护盾、前三次暴击。
- 开战前最多选择 1 个技能，进入战斗后立即消耗。
- 旧版 12 张卡牌数据可自动迁移到新版卡牌 ID。

## 本地运行

需要 Node.js 18 或更高版本。

```bash
npm install
npm start
```

启动后打开终端显示的本地地址。开发命令与正式启动命令相同。

## 在扣子部署

可直接上传本项目压缩包，或让扣子连接 GitHub 仓库。推荐用 GitHub 作为长期维护源，部署配置如下：

- 安装命令：`npm install`
- 启动命令：`npm start`
- 运行时：Node.js 18+
- 入口由 `.coze` 中的配置提供

不要把 `node_modules` 放入压缩包；部署平台会根据 `package.json` 自动安装依赖。

## 数据说明

游客学习数据目前保存在浏览器 `localStorage` 中，同一设备、同一浏览器可持续使用。跨设备同步所需的 Supabase 服务端代码和数据库结构已经保留在 `server/`，但需要在部署平台配置真实的 Supabase 环境变量并完成线上联调后才能启用；当前版本不应把“登录”视为已经完成跨设备同步。

卡牌数量、技能、钻石、逐词记忆进度、每日计划和战斗记录均已纳入本地 v5 数据结构。每日任务按设备本地日期重置。

## 主要文件

- `index.html`：游戏主页面与基础逻辑。
- `assets/cards-data-v5.json`：1629 张标准 JSON 单词卡数据。
- `js/game-upgrades.js`：分页、任务、重复卡、技能和新版战斗逻辑。
- `js/learning-core.js`：可测试的间隔复习、每日计划和地图统计规则。
- `js/learning-system.js`：学习界面、答题队列、旧存档迁移和奖励衔接。
- `js/api.js`、`js/auth.js`：服务端接口与登录界面。
- `server/`：Express/Supabase 服务端与数据库结构。
- `.coze`：扣子项目启动配置。

## 验证

运行 `npm test` 可验证重复计数、地图推进、复习间隔、答错重排和每日计划逻辑。
